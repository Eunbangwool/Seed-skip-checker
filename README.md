# 콩 결주감지기 개조 프로젝트

기존 상용 제품(농기계팜 SW2024-H1, 리셀러 레인메이커코리아, 130만원)이 **결주 아닌데 시도때도
없이 결주로 오탐**하는 문제를 고친다. 씨앗 **계수는 정확**하므로 센서는 정상 → 기존 장치의
**결주 판정 로직이 문제**. 그래서 기존 판정을 버리고 **센서 원신호를 직접 읽어 새로 판정**한다.
기존 펌웨어·회로는 건드리지 않고 **병렬로 얹기만** 한다(non-invasive).

## 방식 (덧대기 / non-invasive)

```
[기존 감지기]                              [새로 붙이는 것]              [자율주행 태블릿]
 4개 광센서 신호선 ──(병렬 탭)──▶  ESP32 (분압/보호회로 → GPIO) ──BLE──▶  네이티브 앱
 씨앗마다 깨끗한 펄스(계수 정확)     간격비율로 진짜 결주만 판정         플로팅 창·큰알람·GPS 결주좌표 기록
```

- **탭 지점 = 4개 광센서의 신호선**(그 컬러 배선 다발). LED/부저 출력이 아니다 —
  LED/부저는 기존 장치의 *틀린 판정 결과*라 탭해봐야 오탐만 옮겨질 뿐.
- 센서 신호선에 **고임피던스로 병렬 연결** → 기존 장치 동작에 영향 없음, A/S 유지, 원상복구 가능
- 통신비/SIM/WiFi 불필요 (캐빈 내 BLE)
- 부품값 1~2만원 (ESP32 + 저항 몇 개 + 배선)

## 결정된 사항
- **탭 지점: 4개 광센서 신호선**(원신호). 기존 LED/부저 출력 아님.
- 판정: 고정시간 아님 → **간격 비율(이동중앙값 대비 배율)**, 땅바퀴 구동이라 간격 일정해 잘 맞음
- 디스플레이: **자율주행 안드로이드 태블릿**(별도 폰 불필요), 플로팅 창(오버레이) 형태
- RTK 위치로 **결주 지점 GPS 좌표 기록 → 재파종**
- 사용자: 납땜·테스터 가능

## 단계
1. **센서 신호선 측정** ← 현재 단계 (`docs/01_신호측정.md`)

## ★진짜 문제 (진행 중 확정)
알림방식이 아니라 **오탐(false positive)**. 결주 아닌데 시도때도 없이 결주 알림.
- 센서 계수는 정상 → 센서·청소 문제 아님 (원인 B 확정)
- 원인: 기존 펌웨어가 **고정시간(watchdog)** 으로 결주 판단 → 땅바퀴 구동이라 속도변하면 씨앗 간 *시간*이 변해 오탐. 설정으로 안 바뀜.
- 해결: 센서 원신호를 직접 읽어 **간격 비율(이동중앙값 대비 배율)** 로 판단. 2배≈1개빠짐, 큰 배율은 선회/정지로 무시.

## 만든 것
- `firmware/kong_gyeolju/kong_gyeolju.ino` — ESP32 펌웨어(4채널 원신호 읽기·간격비율 결주판단·BLE 전송). 측정 후 `ACTIVE_EDGE`/`USE_PULLUP` 2줄만 확정.
- `app/index.html` — 안드로이드 Web Bluetooth 웹앱(연결·채널별 계수·전체화면 결주알람+소리+진동·기록·CSV·화면켜둠).
- `docs/02_배선_인터페이스.md` — 배선·분압저항·핀 배정.
