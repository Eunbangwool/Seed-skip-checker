# 콩 결주감지기 개조 프로젝트

기존 상용 제품(레인메이커코리아 "콩파종기 결주알림시스템 트랙터 부착형", 130만원)의
**알림 방식만** 개선한다. 감지 성능은 이미 충분하므로 **기존 펌웨어는 절대 건드리지 않는다.**

## 방식 (덧대기 / non-invasive)

```
[기존 감지기]                         [새로 붙이는 것]                 [안드로이드 폰]
 LED(또는 부저) 단자 ──전선──▶  ESP32 (분압/보호회로 → GPIO) ──BLE──▶  앱
 결주 시 신호 발생              엣지 감지·디바운스·카운트             진동+큰소리 알림, 횟수·시각 기록
```

- 사용자가 **부저·LED 전선을 이미 뽑아둠** → 그 빈 단자에 원래 컨트롤러가 여전히 결주 신호를 보냄 → **완벽한 탭 지점**
- 기존 회로 무손상, A/S 유지, 실패 시 원상복구 가능
- 통신비/SIM/WiFi 불필요 (캐빈 내 BLE)
- 부품값 1~2만원 (ESP32 + 저항 몇 개 + 배선)

## 결정된 사항
- 탭 지점: **LED 신호선 우선** (부저는 톤/PWM일 수 있어 읽기 까다로움. LED는 깔끔한 ON/OFF)
- 폰: **안드로이드**
- 사용자: 납땜·테스터 가능

## 단계
1. **신호 측정** ← 현재 단계 (`docs/01_신호측정.md`)
2. ESP32 펌웨어
3. 안드로이드 앱

## ★진짜 문제 (진행 중 확정)
알림방식이 아니라 **오탐(false positive)**. 결주 아닌데 시도때도 없이 결주 알림.
- 센서 계수는 정상 → 센서·청소 문제 아님 (원인 B 확정)
- 원인: 기존 펌웨어가 **고정시간(watchdog)** 으로 결주 판단 → 땅바퀴 구동이라 속도변하면 씨앗 간 *시간*이 변해 오탐. 설정으로 안 바뀜.
- 해결: 센서 원신호를 직접 읽어 **간격 비율(이동중앙값 대비 배율)** 로 판단. 2배≈1개빠짐, 큰 배율은 선회/정지로 무시.

## 만든 것
- `firmware/kong_gyeolju/kong_gyeolju.ino` — ESP32 펌웨어(4채널 원신호 읽기·간격비율 결주판단·BLE 전송). 측정 후 `ACTIVE_EDGE`/`USE_PULLUP` 2줄만 확정.
- `app/index.html` — 안드로이드 Web Bluetooth 웹앱(연결·채널별 계수·전체화면 결주알람+소리+진동·기록·CSV·화면켜둠).
- `docs/02_배선_인터페이스.md` — 배선·분압저항·핀 배정.
