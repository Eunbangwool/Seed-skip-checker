# 2단계: ESP32 ↔ 센서 배선 인터페이스

기존 장치는 **그대로 둔다.** ESP32는 센서 신호선에 **병렬로 살짝 얹기만** 한다(고임피던스 입력이라
기존 장치 동작에 영향 없음). 센서 4개 신호선 + 공통 GND만 있으면 된다.

## 반드시 지킬 것
- **ESP32 GPIO는 3.3V까지만 안전.** 신호가 5V/12V면 분압저항 필수.
- **GND 공통 연결** 필수: 기존 장치 GND ↔ ESP32 GND. (안 하면 신호 기준이 안 맞아 오동작)
- ESP32 전원은 기존 장치 5V에서 따와도 되고(전류 여유 확인), USB 보조배터리로 줘도 됨.

## 측정값 → 배선 결정표

1단계에서 잰 **신호선 대기전압 / 콩 지날 때 전압**으로 아래 중 하나를 고른다.

| 신호 형태 (측정 결과) | 배선 | 펌웨어 설정 |
|---|---|---|
| **오픈컬렉터**: 평소 3.3V 근처, 콩 지날 때 0V로 뚝 (또는 반대로 0↔뜸이 불안정) | 분압 불필요. GPIO ↔ 신호선 직결, `INPUT_PULLUP` 사용 | `USE_PULLUP=true`, `ACTIVE_EDGE=FALLING` |
| **푸시풀 3.3V**: 0V ↔ 3.3V 로 깔끔 | 직결 | `USE_PULLUP=false`, 극성대로 EDGE |
| **푸시풀 5V**: 0V ↔ 5V | 분압 **1kΩ(위)+2kΩ(아래)** → 3.3V | 극성대로 EDGE |
| **푸시풀 12V**: 0V ↔ 12V | 분압 **10kΩ(위)+3.9kΩ(아래)** → ≈3.4V | 극성대로 EDGE |

> 분압 회로: `신호선 ── R1 ──┬── GPIO`, `┴── R2 ── GND`. GPIO는 R1·R2 사이에서 딴다.

**극성(EDGE) 정하기**
- 콩(씨앗)이 지날 때 신호가 **HIGH로 올라감** → 씨앗 = 상승엣지 → `ACTIVE_EDGE=RISING`
- 콩이 지날 때 신호가 **LOW로 떨어짐**(오픈컬렉터 대부분) → 씨앗 = 하강엣지 → `ACTIVE_EDGE=FALLING`

## ESP32 핀 배정 (기본값, 펌웨어와 일치)
| 채널 | 파종 골 | ESP32 GPIO |
|---|---|---|
| 1 | 1번 | GPIO 25 |
| 2 | 2번 | GPIO 26 |
| 3 | 3번 | GPIO 32 |
| 4 | 4번 | GPIO 33 |
| GND | 공통 | GND |

(25/26/32/33은 인터럽트 가능·부팅 제약 없는 안전한 핀)

## 배선 순서
1. 기존 장치 전원 끄기.
2. 각 센서 신호선에서 ESP32 해당 GPIO로 (필요시 분압 거쳐) 연결.
3. GND 공통 연결.
4. ESP32 전원 인가 후, 콩을 손으로 하나씩 통과시키며 채널별 계수가 오르는지 시리얼 모니터로 확인.
