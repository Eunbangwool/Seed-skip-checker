# 1단계: 센서 신호선 측정 가이드

목표: **광센서(씨앗 감지) 신호선**이 씨앗 통과 때 내보내는 신호의 **전압**과 **극성**을 알아낸다.
이 값으로 ESP32 입력 보호회로(분압저항)와 펌웨어의 `ACTIVE_EDGE`/`USE_PULLUP`을 정한다.
**ESP32 GPIO는 3.3V까지만 안전**하다.

> ⚠️ 탭 지점은 **광센서 신호선**이다. 기존 장치의 LED/부저 출력이 아니다.
> LED/부저는 기존 장치의 *결주 판정 결과*라서, 그걸 탭하면 지금 문제인 **오탐이 그대로 옮겨진다.**
> 우리는 판정 이전의 **씨앗 원신호**(계수는 정확함이 확인됨)를 읽어 새로 판정한다.

## 어디를 재나
기판 우하단으로 들어오는 **센서 케이블(컬러 배선 다발)**. 채널(골) 1개당 보통 3선:
전원(V+) · GND · **신호(OUT)**. 이 중 **신호선**을 찾는 게 목표다.

## 준비물
- 디지털 테스터(멀티미터)
- 악어클립 또는 가는 점퍼선

## 측정 순서

### A. GND(기준선) 찾기
1. 테스터를 DC 전압(V⎓) 20V 범위로.
2. 검은 프로브(COM)를 장치 **전원 마이너스/GND**에 고정. (모든 측정의 기준)

### B. 한 센서의 3선 중 신호선 찾기 ★핵심★
1. 장치 전원 ON(정상 상태).
2. 빨간 프로브를 센서 커넥터 각 핀에 하나씩 대본다.
   - 항상 일정한 높은 전압(예: 5V/12V) → **전원(V+)**
   - 0V 근처 → GND
   - 나머지 한 선이 **신호(OUT)** 후보
3. 신호 후보 핀에 프로브 고정하고, **콩(또는 손가락)을 센서 구멍 사이로 천천히 통과**시킨다.
   - 통과 순간 전압이 **툭 바뀌면 = 신호선 확정**
   - 올라감(0→높음) = 씨앗 통과 시 HIGH → `ACTIVE_EDGE=RISING`
   - 떨어짐(높음→0) = 씨앗 통과 시 LOW → `ACTIVE_EDGE=FALLING` (광센서 대부분 이쪽)

## 기록표 (채워서 알려주세요)

| 항목 | 값 |
|---|---|
| 전원선(V+) 전압 | ___ V |
| 신호선 대기 전압 | ___ V |
| 신호선 콩 통과 시 전압 | ___ V |
| 극성 | 올라감(RISING) / 떨어짐(FALLING) |
| 4채널 배선이 서로 같은 형태인가 | 예 / 아니오 |

이 값으로 → 분압저항(3.3V 초과 시)과 펌웨어 2줄을 확정한다. 자세한 배선은 `docs/02_배선_인터페이스.md`.

## 안전 주의
- 프로브로 두 핀을 **동시에 잇지 말 것**(쇼트).
- 신호가 5V/12V면 ESP32에 직결 금지 → 분압저항 필수(값은 `docs/02` 표 참고).
- **GND 공통 연결** 필수: 기존 장치 GND ↔ ESP32 GND.
